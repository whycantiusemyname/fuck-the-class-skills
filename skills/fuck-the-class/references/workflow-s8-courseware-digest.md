# S8 课件消化

目的：把章节课件编译成 `20_知识/` 中的三层学习材料：给学生建立第一轮 mental model 的 `主干讲解`，给 AI tutor 控制边界的 `grounding`，以及给学生进入 S10 问答/做题的 `S10启动卡`。

S8 的第一优先级是高可读性的学生主干讲解。grounding 和 S10启动卡都是辅助层，不得替代主干讲解。若时间或上下文只够生成一个文件，优先生成 `主干讲解.md`。

## 输入

- 课程根目录
- 章节范围和对应课件列表
- 可选根目录 `课程口径.md`；缺失时以给定课件为边界
- 用户确认的主线/补充映射，如“6-5 补充 6-4，不是主线”。这来自教师重点和考试提示，不能自行推断；缺失时询问或标 `范围待确认`
- 可选 `10_题库/_标签库.md` 和 `40_派生视图/考频矩阵.md`，只用于命名题型与标注优先级

## 模式

- `默认`：生成或更新 `主干讲解.md`，并尽量生成 `grounding.md` 与 `S10启动卡.md`。
- `grounding-only`：只有用户明确要求只做 AI tutor 边界材料时使用。
- `quick-start-only`：只有用户明确要求只做 S10 快速启动卡时使用。
- `补全`：针对已有章节材料补来源、章节承接、公式解释、题型起手、主动 probe 或范围待确认。
- `速查`：只有用户明确要求“速查卡 / 考前卡片 / 极简卡”时才生成；速查卡不能替代主干讲解。

旧文件名兼容但不作为新默认：`第XX章_入门讲解.md` 视为旧版启动卡，`第XX章_主干重点.md` 视为旧版主干讲解。新生成默认使用 `主干讲解`、`grounding`、`S10启动卡`。

## 必需文件

- 读取：源 PPT/PPTX/PDF 课件，最好在 `00_原材料/`
- 读写或创建：`20_知识/README.md`
- 若存在且受影响，读写：现有章节笔记和 `20_知识/整体知识框架.md`
- 可选读取：`10_题库/_标签库.md` 和 `40_派生视图/考频矩阵.md`
- 默认写入：`20_知识/第XX章_主题_主干讲解.md`
- 建议写入：`20_知识/第XX章_主题_grounding.md`、`20_知识/第XX章_主题_S10启动卡.md`
- 可选写入：`20_知识/第XX章_主题_速查卡.md`
- 必要时写转换 cache 到 `90_缓存/`

## 输出原则

1. `主干讲解.md`
   - 给学生读，用来建立第一轮 mental model。
   - 必须有章节承接、整体直觉、核心概念解释、公式意义、题型起手、易错点、复习检查。
   - 不是教材逐页转写，也不是速查表、关键词列表或连续 bullet。
   - 没有主干讲解时，不得宣称默认 S8 完成。
2. `grounding.md`
   - 给 AI tutor 用，写来源、边界、课程用语、公式方法、题型信号、S10 probe 种子和范围待确认。
   - 可以结构化、短句化，不追求学生阅读体验。
   - 不得替代主干讲解。
3. `S10启动卡.md`
   - 给学生进入 S10 前快速启动。
   - 必须从主干讲解压缩而来，开头声明“不替代主干讲解”。
   - 只写快速地图、题型分流、第一动作、最小自测和提问入口。
   - 不得用启动卡替代主干讲解。

生成顺序：先生成或更新 `主干讲解.md`，再生成 `grounding.md`，最后生成 `S10启动卡.md`。

## 主干讲解结构

`主干讲解.md` 默认使用下面结构。标题可加章节名，但不得改成速查式短标题。

- `本章放在课程里的位置`：用 2-4 段说明前置章节解决什么、本章换了什么问题、本章为什么自然出现。
- `这一章抓什么`：用 3-6 段讲本章主问题，不只列重点。
- `先建立整体直觉`：用连续故事解释核心机制，说明能量/变量从哪里来、怎样变化、哪些条件维持变化、哪些元件决定时间尺度/幅度/频率。
- `本章怎么串起来`：用文字、表格、箭头链或 Mermaid 说明概念关系；不能只列概念名。
- 核心主题若干节：每节必须包含 `直觉`、`公式 / 条件`、`公式为什么这样`、`题目怎么起手`、`易错点`。
- `典型题型入口`：每类题写题面信号、第一动作、为什么是这个第一动作、常见误判。
- `本章总图`：用表格或流程图总结概念、判断信号、关键公式、第一步和常见坑。
- `复习检查`：用 checkbox 或问题清单，每项回指对应小节。
- `进入 S10 的问题入口`：列 5-8 个适合进入问答的问题，让主干讲解自然接到 S10。

每个核心主题如果只有公式和列表，判为不合格。公式必须服务理解：紧邻说明符号含义、使用条件、题目触发信号或推导关系。

## Grounding 结构

`grounding.md` 建议结构：

- `来源与边界`: 源课件、页码、主线/补充/排除。
- `本章主线`: 本章知识如何串起来。
- `课程用语与记号`: 只写本课使用的叫法和符号。
- `公式 / 方法 / 条件`: 每个非平凡公式、判据、方法都能追溯来源。
- `题型信号与第一步`: 优先使用 `_标签库.md` 的题型名。
- `高频/考试提示`: 来自 S2 或题库证据；没有证据就写“未统计”。
- `S10 主动 probe 种子`: 让学生暴露理解缺口的小问题。
- `范围待确认`: 无法追溯但可能相关的内容。

## S10 启动卡结构

`S10启动卡.md` 开头必须写：

```markdown
> 这是进入 S10 问答/做题前的启动卡，不替代 `主干讲解.md`。
> 完全没学过本章时，先读主干讲解；读过但准备做题时，再读这份。
```

正文只保留：

- 5 分钟地图
- 题型分流
- 第一动作
- 最小自测，每题附参考答案、判断标准或答案要点
- 提问入口

S10启动卡不追求完整讲懂；它是从主干讲解进入互动学习的入口。

## 格式纪律

- 主干讲解以解释性段落为主。bullet 只用于总结、步骤、易错点、检查表和入口清单。
- 单段通常控制在 80-180 个中文字符；单段超过 250 个中文字符会触发 lint warning。
- 连续 bullet 超过 8 行会触发 lint warning；主体 bullet 占比超过 45% 会触发 hard failure。
- 每个 bullet 组前必须有解释段说明它为什么重要。
- 单行公式用 inline LaTeX；多行推导、核心公式或结构标注用 display block。
- 推荐使用 callout、`\boxed{}`、`aligned`、`\underbrace{}`、过程链和对照表来服务理解。
- 颜色只用于强调；任何重点必须同时用文字、加粗、callout 或公式结构表达，不能只靠颜色。
- 每章主干讲解至少应有 1 个因果链、状态循环或过程链。Mermaid 可作为增强，不能替代文字解释。

## 来源与范围 gate

1. 课件通过 `pdf-ingestion.md` 在 `90_缓存/` 下做缓存转换；不要操控用户打开的 Office/WPS 窗口。
2. 先做范围审计：列出源课件、已确认主线/补充关系、明确排除内容和未确认项。未确认项影响可考边界时，不写成必会内容。
3. 范围优先级：当前用户指令、`课程口径.md` 确认范围、用户确认教师重点、源课件。
4. `学习阶段` 只决定术语熟悉度、先修展开和推理桥数量，不授权新增内容。
5. 不用更强 formulation 替换课件表述，不引入材料外记号、证明工具、术语或方法。
6. 普通类比可以解释范围内内容，但不能添加新定理、记号系统、证明方法或可考结论。
7. 补充或非主线内容放在 `课件拓展（非应试主线）`，不要写成必会。
8. 题库和标签库只能帮助命名与优先级，不能授权材料外理论。

## README 与框架

- 章节到课件映射表只放在 `20_知识/README.md`，不要在章节 note 中重复整表。
- 章节内引用到课件文件名；页码只在能定位实际内容时写，禁止用“第 1-总页数 页”伪范围。
- 每个非平凡公式、判据、命名方法都能追溯到来源段落。
- 改章节 note 后检查 README 并更新；无需更新时报告已检查且未变。
- 若 `20_知识/整体知识框架.md` 已存在或用户要求，刷新受影响部分。不存在且用户未要求时不隐式创建。

## 校验

1. 校验前做范围审计，并报告 `范围复核：课件外理论 0，未确认术语 0`，或列出未解决项。未解决范围项阻塞完成，或必须留在 `范围待确认`。
2. 运行：

```text
python <skill>/scripts/lint_s8_chapter_docs.py --knowledge-root <course-root>/20_知识
python <skill>/scripts/validate_course_artifacts.py --course-root <course-root> --scope s8
```

3. 用每个源课件/PDF、每个适用 PDF completion certificate、每个改动后的主干讲解/grounding/S10启动卡/README/framework 运行 `s8_digest_gate.py bind`。intentional refresh 时使用 `--replace`。若用户明确要求单独产物，传入 `--mode grounding-only` 或 `--mode quick-start-only`。最后运行 `s8_digest_gate.py verify`。

## 输出

默认：

- `20_知识/第XX章_主题_主干讲解.md`
- `20_知识/第XX章_主题_grounding.md`
- `20_知识/第XX章_主题_S10启动卡.md`
- 已同步或确认未变的 `20_知识/README.md`

可选：

- `20_知识/第XX章_主题_速查卡.md`
- `20_知识/整体知识框架.md`

## 边界

不发明练习题，不导入课外理论。题目入库归 S1，完整解答归 S9。不要粘贴课件长篇逐字内容。scope 或来源冲突未解决、README/框架仍过期、主干讲解 lint 未通过、digest manifest 缺失或 stale 时，不报告默认 S8 完成。S8 的默认产物服务 S10 主线程问答，但主干讲解不是学习终点：它应通向 S10 的做题、追问、纠错和变式验证。
