# S8 课件消化

目的：把章节课件转成 `20_知识/` 中的 **AI tutor grounding + 学生最小启动入口**。S8 的默认目标不是写一篇越长越好的静态讲义，而是让 S10 能在本课程口径内快速讲一轮、主动 probe、进入做题和纠错循环。

完整主干讲义仍可生成，但只在用户明确要求“完整讲义 / 主干重点长文档 / 系统讲完这一章”时作为可选模式。

## 输入

- 课程根目录
- 章节范围和对应课件列表
- 可选根目录 `课程口径.md`；缺失时以给定课件为边界
- 用户确认的主线/补充映射，如“6-5 补充 6-4，不是主线”。这来自教师重点和考试提示，不能自行推断；缺失时询问或标 `范围待确认`
- 可选 `40_派生视图/考频矩阵.md` 用于标注高频题型

## 模式

- `启动`：默认。生成 grounding、启动卡、主动 probe 种子和条件/坑点/题型信号。
- `完整`：用户明确要求完整讲义时，生成较长 guided review。
- `补全`：针对已有章节笔记补来源、题型信号、主动 probe、条件/坑点。

## 必需文件

- 读取：源 PPT/PPTX/PDF 课件，最好在 `00_原材料/`
- 读写或创建：`20_知识/README.md`
- 若存在且受影响，读写：现有章节笔记和 `20_知识/整体知识框架.md`
- 可选读取：`10_题库/_标签库.md` 和 `40_派生视图/考频矩阵.md`
- 默认写入：`20_知识/第XX章_启动卡.md` 和/或 `20_知识/第XX章_grounding.md`
- 可选写入：`20_知识/第XX章_主干重点.md`
- 必要时写转换 cache 到 `90_缓存/`

## 步骤

1. 课件通过 `pdf-ingestion.md` 在 `90_缓存/` 下做缓存转换；不要操控用户打开的 Office/WPS 窗口。
2. 先做范围审计：列出源课件、已确认主线/补充关系、明确排除内容和未确认项。未确认项影响可考边界时，不写成必会内容。
3. 默认 `启动` 模式输出两个层次：
   - **AI tutor grounding**：给 S10 用，包含课程语言、来源页、主线关系、公式/方法、考试边界、高频题型、不要引入的课外内容。
   - **学生最小启动卡**：给学生用，只建立最低必要认知地图，让学生从“无问题状态”进入“能做第一步/能追问/能暴露错误”的状态。
4. 启动卡必须回答：
   - 这一章解决什么类型的问题？
   - 看到题目先看什么信号？
   - 第一动作是什么？
   - 公式/方法成立条件是什么？
   - 最容易和什么混淆？
   - 立刻进入 S10 的最小 probe 是什么？
5. AI tutor grounding 建议结构：
   - `来源与边界`: 源课件、页码、主线/补充/排除。
   - `本章主线`: 本章知识如何串起来。
   - `课程用语与记号`: 只写本课使用的叫法和符号。
   - `公式 / 方法 / 条件`: 每个非平凡公式、判据、方法都能追溯来源。
   - `题型信号与第一步`: 优先使用 `_标签库.md` 的题型名。
   - `高频/考试提示`: 来自 S2 或题库证据；没有证据就写“未统计”。
   - `S10 主动 probe 种子`: 让学生暴露理解缺口的小问题。
   - `范围待确认`: 无法追溯但可能相关的内容。
6. 学生启动卡建议结构：
   - `这章一句话`
   - `你先只需要会的三件事`
   - `看到题先做什么`
   - `最容易错的地方`
   - `一个 30 秒 probe`
   - `做题前不用急着懂的补充内容`
7. 如果用户要求 `完整` 模式，再生成 `第XX章_主干重点.md`。完整讲义可以有较长 guided review，但仍要服务 S10，而不是替代互动学习。完整模式结构：
   - `这一章抓什么`
   - `阅读层次表`
   - `先建立整体直觉`
   - `本章怎么串起来`
   - `分节主干`
   - `主动问答种子`
   - `条件 / 坑点 / 反例 / 题型信号`
   - `解题流程`
   - `易错点`
   - `复习检查`
8. 来源纪律：
   - 章节到课件映射表只放在 `20_知识/README.md`，不要在章节 note 中重复整表。
   - 章节内引用到课件文件名；页码只在能定位实际内容时写，禁止用“第 1-总页数 页”伪范围。
   - 每个非平凡公式、判据、命名方法都能追溯到来源段落。
   - 题库和标签库只能帮助命名与优先级，不能授权材料外理论。
9. 范围 gate：
   - 范围优先级：当前用户指令、`课程口径.md` 确认范围、用户确认教师重点、源课件。
   - `学习阶段` 只决定术语熟悉度、先修展开和推理桥数量，不授权新增内容。
   - 不用更强 formulation 替换课件表述，不引入材料外记号、证明工具、术语或方法。
   - 普通类比可以解释范围内内容，但不能添加新定理、记号系统、证明方法或可考结论。
   - 补充或非主线内容放在 `课件拓展（非应试主线）`，不要写成必会。
10. 格式纪律：
    - 单行公式用 inline LaTeX；多行推导、核心公式或需要标注结构时用 display block。
    - 段落适合电子阅读。一个段落承担多个功能时拆开；长段必须拆。
    - 启动卡应短而可行动；不要把完整讲义压进启动卡。
11. 维护 `20_知识/README.md`：章节到课件映射、主线/补充层级、建议先读顺序。改章节 note 后检查 README 并更新；无需更新时报告已检查且未变。
12. 维护 cross-chapter framework：若 `20_知识/整体知识框架.md` 已存在或用户要求，刷新受影响部分。不存在且用户未要求时不隐式创建。
13. 校验前做范围审计，并报告 `范围复核：课件外理论 0，未确认术语 0`，或列出未解决项。未解决范围项阻塞完成，或必须留在 `范围待确认`。
14. 运行 `python <skill>/scripts/validate_course_artifacts.py --course-root <course-root> --scope s8`。
15. 用每个源课件/PDF、每个适用 PDF completion certificate、每个改动后的 grounding/启动卡/主干笔记/README/framework 运行 `s8_digest_gate.py bind`。intentional refresh 时使用 `--replace`。最后运行 `s8_digest_gate.py verify`。

## 输出

默认：

- `20_知识/第XX章_grounding.md`
- `20_知识/第XX章_启动卡.md`
- 已同步或确认未变的 `20_知识/README.md`

可选：

- `20_知识/第XX章_主干重点.md`
- `20_知识/整体知识框架.md`

## 边界

不发明练习题，不导入课外理论。题目入库归 S1，完整解答归 S9。不要粘贴课件长篇逐字内容。scope 或来源冲突未解决、README/框架仍过期、digest manifest 缺失或 stale 时，不报告 S8 完成。S8 的默认产物服务 S10 主线程问答；完整讲义不是默认主学习场。
