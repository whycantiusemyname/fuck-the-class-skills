# S0 课程初始化/检查

目的：初始化或审计一门课的文件夹，使后续 S1-S10 都能找到预期文件。S0 完成后，主线程默认进入 S10 私教 runtime 姿态。

## 输入

- 课程根目录
- 可选学科名
- 学习阶段，如 `大一`、`大二下`、`研一`；用户跳过时使用 `未设置`
- 可选已确认教学/考试范围、教师重点、主线/补充关系、明确排除内容

## 必需文件

- 除课程根目录外无

## 步骤

1. 按 `schema-and-rules.md` 创建缺失目录。
2. 若 `课程口径.md` 缺失，询问学习阶段和可选范围字段，然后运行 `course_profile.py init`。跳过的阶段写 `未设置`，空白可选字段写 `未设置`。审计已有 profile 时只验证；用户明确要求更新范围时，优先运行 `course_profile.py update`，不要手改结构。
3. 若缺失，创建 `10_题库/_标签库.md`。
4. 若缺失，创建 `30_我的数据/做题记录.md`。
5. 若缺失，创建空的 `30_我的数据/学习事件.jsonl`。
6. 若缺失，创建 `30_我的数据/卡点清单.md`。
7. 保持 `40_派生视图/` 为空；只有 S2/S4/S5/S6/S10 生成派生文件。
8. 运行 `course_profile.py verify`。
9. 报告新建项、已存在项、当前课程边界 fallback，并说明后续自然进入 S10 主线程私教 runtime。

## 输出

- 初始化后的课程骨架
- 根目录 `课程口径.md`
- setup/audit 摘要
- S10 runtime 可用性说明

## 边界

除非用户要求，不把原材料移动进 `00_原材料/`；只创建它们应该放置的位置。S0 不生成学习计划、不写派生状态快照、不替用户决定后续学习节奏。
