# S0 Course Setup

Purpose: initialize or audit a subject folder so every later workflow has the files it expects.

Inputs:

- course root
- optional subject name
- learning stage as free text, such as `大一`, `大二下`, or `研一`; if the user skips it, use `未设置`
- optional confirmed teaching/exam scope, teacher emphasis and mainline/supplement decisions, and explicit exclusions

Required files:

- none beyond the course root

Steps:

1. Create the full directory tree from `schema-and-rules.md` if missing.
2. If `课程口径.md` is missing, ask for the learning stage and the optional range fields above, then run `course_profile.py init`. A skipped stage becomes `未设置`; blank optional fields become `未设置`. During an audit, validate an existing profile but never overwrite it without an explicit update request.
3. Create `10_题库/_标签库.md` if missing.
4. Create `30_我的数据/做题记录.md` if missing.
5. Create `30_我的数据/卡点清单.md` if missing.
6. Leave `40_派生视图/` empty except for files generated by S2/S4/S5/S6.
7. Run `course_profile.py verify` and report created items, existing items, and the active course-boundary fallback separately.

Output:

- initialized course folder skeleton
- root-level `课程口径.md`
- setup/audit summary

Boundary: do not move raw materials into `00_原材料/` unless the user asks; just create the place where they belong.
