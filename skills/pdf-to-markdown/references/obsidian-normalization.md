# Obsidian Normalization Rules

Use minimal compatibility cleanup only. Do not turn this stage into open-ended rewriting or heuristic repair. The final segmented formatting pass belongs to the later LLM readthrough, not to these deterministic scripts.

## Required Output Shape

- single-chunk mode: one exported `full.md` plus sibling `images/`
- merge mode: one Markdown chunk per successful MinerU chunk
- merge mode: one merged Markdown draft
- merge mode: one asset directory at `assets/<note-stem>/`
- one lightweight metadata file per chunk plus one export or merge metadata file
- one later segmented LLM readthrough package generated by the main runner

## Image Rules

- Copy image assets into `assets/<note-stem>/`
- Prefix renamed assets with the chunk page range
- If MinerU output already uses Obsidian embeds, preserve them
- Only rewrite image references when the source still uses Markdown or HTML image syntax
- If an image width cap is provided, apply `|WIDTH` only to images wider than that cap

These rules apply only in merge mode. In single-chunk mode, keep MinerU `full.md` and `images/` together and do not rewrite links.

## Text Rules

- Normalize line endings to `\n`
- Preserve whitespace and heading structure otherwise

These rules apply only in merge mode. In single-chunk mode, export the original `full.md` text unchanged.

## Math Rules

- Repair `\(...\)` to `$...$`
- Repair `\[...\]` to `$$ ... $$`
- Repair clearly math-like `[ ... ]` blocks to `$$ ... $$`
- Do not rewrite formula content beyond delimiter repair
- Do not attempt semantic LaTeX correction or table restructuring
- Do not try to detect or classify every complex math problem in script code; let the later segmented LLM pass read and repair the Markdown directly

## Detection Policy

- Do not generate heuristic issue reports
- Do not guess chunk-boundary joins
- Do not auto-detect caption problems
- Metadata files should stay lightweight and support the later LLM readthrough package, not replace it
